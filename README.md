# Drum Pattern Trainer Prototype

PyQt5 で作った、ドラム練習用のプロトタイプです。

ユーザが左側のグリッドでお手本パターンを作成し、右側で実際の入力を確認しながら、MIDI ドラムパッドやキーボードで反復練習できます。

## 主な機能

- 16 分グリッドでのお手本パターン入力
- 左: お手本 / 右: 実入力 の 2 画面構成
- MIDI ドラムパッド入力対応
- キーボード入力対応（簡易テスト用）
- レーン順の並べ替え
- キャリブレーション（Kick 4 つ打ち）
- 手動補正値の設定
- 白線ガイドの ON/OFF と間隔設定（4 分 / 8 分 / 16 分）
- お手本音 / 入力音 / メトロノーム音の切り替え
- 音源の zip / フォルダ読み込み
- お手本編集時にノートを置くと対応する音を再生

## 対応レーン

標準では以下の 5 レーンを使います。

- Kick
- Snare
- Hi-Hat
- Tom
- Crash

## 動作環境

- Python 3.9 以上推奨
- Windows での利用を主に想定
- PyQt5
- mido

環境によっては MIDI 入力まわりで追加のバックエンドが必要になることがあります。

## インストール

```bash
pip install PyQt5 mido
```

必要に応じて、MIDI バックエンドも導入してください。

例:

```bash
pip install python-rtmidi
```

## 起動方法

リポジトリを clone したあと、対象ファイルを実行してください。

```bash
python drum_trainer_pyqt_edit_sound_guides_audio_light.py
```

ファイル名は手元の版に合わせて読み替えてください。

## 画面構成

### 上部バー
- BPM
- 小節数
- 再生 / 停止
- キャリブレーション開始
- 音源 zip 読込
- 音源フォルダ読込
- メトロノーム ON/OFF
- お手本音 ON/OFF
- 入力音 ON/OFF
- MIDI 更新
- MIDI 入力ポート選択
- 手動補正値
- 白線 ON/OFF
- 白線間隔

### 左側
お手本入力用のグリッドです。

- クリックでノートを配置 / 削除
- ノート追加時に対応する音を再生

### 右側
実入力表示用のグリッドです。

- MIDI パッドやキーボードからの入力を表示
- お手本と比較してズレを確認

## 操作方法

### 1. お手本を作る
左側のグリッドをクリックしてパターンを作成します。

### 2. MIDI デバイスを選ぶ
ドラムパッドを接続し、`MIDI更新` を押してポート一覧を更新してください。

### 3. 再生する
`再生` を押すと、お手本パターンがループ再生されます。

### 4. 叩く
ドラムパッド、またはキーボードで入力します。

### 5. 比較する
右側の実入力表示を見て、タイミングのズレを確認します。

## キーボード割り当て

MIDI 機器がない場合でも、以下のキーで簡易入力できます。

- `A` = Kick
- `S` = Snare
- `D` = Hi-Hat
- `F` = Tom
- `G` = Crash

## キャリブレーションについて

キャリブレーションは、表示タイミングと入力タイミングのズレを測って補正するための機能です。

このプロトタイプでは、現在の BPM に基づく Kick の 4 つ打ちを流し、それに合わせてユーザが Kick を叩きます。そこから補正値を計算し、適用するかどうかを選べます。

また、上部の手動補正欄から ms 単位で直接補正値を設定することもできます。

## 音源の読み込み

音源は以下の 2 方式で読み込めます。

- zip ファイルから読み込み
- フォルダから読み込み

### 推奨ファイル名

- `kick.wav`
- `snare.wav`
- `hihat.wav`
- `tom.wav`
- `crash.wav`
- `metronome_major.wav`
- `metronome_minor.wav`

多少ゆるい名前でも拾えるようにしていますが、上記に近い名前をおすすめします。

### 音源がない場合
内蔵の簡易音で動作します。

## 白線ガイド

再生中の目安として白線を表示できます。

- ON / OFF 切り替え可能
- 間隔は 4 分 / 8 分 / 16 分から選択可能

## レーン順の変更

Kick や Snare などの列順は、レーン順設定から変更できます。

変更は左のお手本 UI と右の実入力 UI の両方に反映されます。

## 既知の注意点

- プロトタイプのため、UI や内部実装はまだ安定化の途中です。
- OS や MIDI ドライバ環境によって、起動後のデバイス認識が不安定な場合があります。
- 同じ音の高速連打時は、環境によっては発音が不完全になることがあります。
- リアルタイム性能は PC 性能や音声環境に影響されます。
